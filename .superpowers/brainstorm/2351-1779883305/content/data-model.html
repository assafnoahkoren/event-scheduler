StockLedgerEntry

— the single source of truth for current balance

Every operation writes here atomically. SUM(quantityDelta) per (item, location) = current stock.

itemId FK → StockItem locationId FK → StockLocation quantityDelta float (+ or −) operationType: ARRIVAL | MOVEMENT_IN | MOVEMENT_OUT | RECOUNT_ADJUSTMENT referenceId (links back to the source row)
How balance is computed (Prisma):
prisma.stockLedgerEntry.groupBy({ by: ['itemId','locationId'], _sum: { quantityDelta: true } })
Flow example: Order 50 crates → 40 arrive at warehouse (+40 ledger entry) → 20 moved to bar (−20 warehouse, +20 bar) → recount bar = 17 (−3 adjustment entry). Result: warehouse has 20, bar has 17.
