User Stories — On the Job, Mobile

Four roles, four contexts. Which ones match your team, and are any missing?

🚚
Receiving a delivery
Warehouse worker, truck just arrived
  1. Open app → see open purchase orders
  2. Tap the relevant order
  3. Enter how many actually arrived (may be less than ordered)
  4. Confirm which location it goes to
  5. Done — order auto-updates to PARTIAL or COMPLETE
⚡ Goal: under 30 seconds, one hand, no typing
🔄
Moving stock to the bar
Setup worker, pre-event
  1. Tap "Move Stock"
  2. Select item (e.g. Beer 330ml)
  3. Use +/− stepper to enter quantity
  4. Confirm From → To locations
  5. Done — ledger updated
⚡ Goal: log while carrying the crates, not after
📋
Post-event recount
Bar manager, after event ends
  1. Tap "Recount" → select location (Bar A)
  2. App shows each item with expected count
  3. Enter actual count per item (stepper)
  4. Skip items that weren't touched
  5. Submit — system records all deltas at once
⚡ Goal: works like a checklist, item by item, no thinking required
👁️
Checking stock before ordering
Event manager, planning next event
  1. Open Stock → Levels
  2. See totals per item across all locations
  3. Identify what needs replenishing
  4. Tap item → Create purchase order
⚡ Goal: immediate answer to "do we have enough X?"
📱 Mobile-first UX principles for all flows:
✅ Large +/− steppers for quantities (no keyboard)
✅ Context-aware: "I'm at Bar A" pre-selects location
✅ Bottom sheet drawers, not new pages
✅ Batch submit for recounts (all items at once)
✅ "Expected vs counted" shown side by side
✅ Quick actions prominent on home screen
